# Release 0.22.0 — MCP Core

Release 0.22.0 makes MCP conversational analysis the default Codex Usage
Tracker experience. A clean plugin installation now exposes exactly 7 focused
core tools; the existing Evidence Console remains available for local
verification and compatibility workflows.

## MCP Profiles

| Profile | Tools | Purpose |
| --- | ---: | --- |
| `core` | 7 | Default conversational analysis surface |
| `full` | 59 tools | 54 non-developer 0.21/PR290 names plus the core adapters |
| `developer` | 64 | Every 0.21/PR290 name plus the core adapters |

The ordered core surface is:

1. `usage_status`
2. `usage_refresh`
3. `usage_analyze`
4. `usage_query`
5. `usage_evidence`
6. `usage_allowance`
7. `usage_job_status`

Every core tool returns `codex-usage-tracker.mcp-envelope.v1`, including an
explicit result schema, scope, freshness, accounting context, messages, and
next actions. The opt-in `developer` profile retains every 0.21/PR290 tool
name; `full` retains every non-developer compatibility name.

## What Changed

- Added canonical query, analysis, evidence, allowance, refresh, status, and
  job application boundaries behind the seven core tools.
- Made `core` the default for repository and installed-plugin MCP launchers.
- Added deterministic routing coverage for ten representative usage questions,
  with a maximum of three analytical calls excluding refresh and job polling.
- Kept canonical totals, tier pricing, allowance intelligence, and subagent
  reporting on their existing calculation paths with equivalence coverage.
- Repositioned the existing browser experience as the supporting Evidence
  Console. No dashboard navigation changed in this release.
